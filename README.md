# EEG-Based Emotion Recognition on DEAP Dataset – A Reproducible Benchmark

This repository contains the complete code and experimental pipeline for our paper submitted to NexSymp 2026:

**"A Benchmark Study of Classical Machine Learning for Cross-Subject EEG-Based Emotion Recognition on the DEAP Dataset"**

We provide a transparent, reproducible cross-subject baseline under leave-one-subject-out (LOSO) validation, including feature extraction (PSD, DE, Hjorth), four classifiers (KNN, SVM, Random Forest, XGBoost), hyperparameter optimisation, statistical evaluation, and SHAP-based model interpretation.

## Key Results

| Dimension | Best Model | Accuracy (mean±std) | F1 (macro mean±std) | AUC |
|-----------|------------|---------------------|----------------------|-----|
| Valence   | Random Forest | 56.02% ± 8.79% | 43.29% ± 9.72% | 0.567 ± 0.138 |
| Arousal   | SVM           | 53.59% ± 14.72%| 41.58% ± 10.40%| 0.537 ± 0.103 |

*Note: Labels are binarised using a threshold of 5 on the 1-9 SAM scale (ratings >5 → positive/high, ≤5 → negative/low). F1 scores are macro-averaged.*

All results are obtained under strict 32-fold LOSO (test on each subject once) with nested 3-fold inner CV for hyperparameter selection.

Install all dependencies using:

```bash
pip install -r requirements.txt
```

> **Note on SHAP**: If you encounter `ValueError: could not convert string to float`, downgrade SHAP to `0.42.1` (`pip install shap==0.42.1`). This is a known compatibility issue with some XGBoost versions.

## Data Preparation

1. Download the DEAP dataset (preprocessed MATLAB version) from the official website.
2. Extract the files into a folder (e.g., `D:\EEG\data_preprocessed_matlab` on Windows, or `/home/user/EEG/data_preprocessed_matlab` on Linux/macOS).
3. Update the `DATA_DIR` variable in `main.py` (and in `feature_extract.py` if needed) to point to your data folder.

## Running the Experiments

All paths in the scripts are currently set to Windows-style `D:\EEG\...`. For cross-platform use, modify the `DATA_DIR`, `RESULTS_DIR`, and `CACHE_DIR` variables in the scripts to your preferred locations (absolute or relative).

### Step 1: Validate the environment and pipeline (optional)

```bash
python check_deap_env.py      # prints Python version and installed packages
python test_small.py          # quick test on two subjects (should run without errors)
```

### Step 2: Run the full LOSO experiment

```bash
python main.py
```

This will:
- Extract features for all 32 subjects (cached in `cache/` for faster re-runs).
- Perform 32-fold LOSO for both valence and arousal tasks.
- Perform PCA (95% variance) and hyperparameter search (3-fold inner CV) inside each fold.
- Save results to `results/results.json` and also save `X_all.npy`, `y_val_all.npy`, `y_aro_all.npy`, `subject_ids.npy` in `results/`.

> **Cache management**: If you modify `feature_extract.py` (e.g., change feature types), you MUST delete the `cache/` folder manually to force re-extraction. Otherwise, old cached features will be reused and results will be incorrect.

> **Expected runtime**: First run (feature extraction + LOSO) ~15–30 minutes on a modern CPU. Subsequent runs skip feature extraction due to caching.

### Step 3: Generate basic paper figures (Figure 2, 3a, 3b)

```bash
python visualize_results.py
```

Outputs saved to `figures/`:
- `figure2_model_performance.png` – bar chart comparing Accuracy and F1 for all models.
- `figure3a_shap_valence.png` – Top-10 SHAP features for valence.
- `figure3b_shap_arousal.png` – Top-10 SHAP features for arousal.

### Step 4: Generate extra analytical figures (optional)

```bash
python generate_extra_figures.py
```

This script requires `mne` (included in `requirements.txt`) and produces:
- `extra_pca_variance.png` – Cumulative explained variance plot.
- `extra_feature_group_comparison.png` – Comparison of PSD, DE, Hjorth, and fusion groups.
- `extra_boxplot.png` – Boxplot of 32-fold accuracies per model.
- `extra_shap_heatmap.png` – Channel-by-band SHAP importance heatmap.
- `extra_topomap.png` – Scalp topography of feature importance.

> **Note**: This script depends on `X_all.npy`, `y_val_all.npy`, etc., which are generated by `main.py`. Run `main.py` first to create these files.

## Reproducing the Paper Results

All numerical results reported in the paper are contained in `results/results.json`. You can load it with:

```python
import json
with open("results/results.json", "r") as f:
    data = json.load(f)
print(data["valence"]["summary"]["RF"]["accuracy"])   # mean and std
```

The exact random seeds, hyperparameter grids, and PCA threshold are hard-coded in `main.py` to ensure reproducibility.

## Interpreting the Output

- **Feature dimension**: Our `feature_extract.py` extracts 352 features per trial (128 PSD + 128 DE + 96 Hjorth).
- **LOSO**: Each of the 32 folds tests on exactly one subject; the training set consists of the other 31 subjects. This is the most challenging cross-subject protocol.
- **SHAP analysis**: The SHAP values reported in the paper and generated by `visualize_results.py` are computed on **the entire dataset** using an XGBoost model (not per-fold). This is done for global interpretability (identifying which EEG channels/frequency bands are most discriminative for emotion) and is **not** meant to assess cross-subject generalisation. The LOSO accuracy results serve that purpose.

## Troubleshooting

- **`shap` error `ValueError: could not convert string to float`** → Downgrade `shap` to 0.42.1 as described above.
- **`MNE` missing** → `pip install mne` (included in `requirements.txt`).
- **Missing `.npy` files in `results/`** → Run `rebuild_npy.py` to regenerate them from the cached `.pkl` files.
- **Out-of-memory / slow performance** → Reduce `n_jobs` in `main.py` (e.g., set `n_jobs=2` in `GridSearchCV` and `XGBClassifier`). The default uses all available cores.
- **MNE topomap fails with channel not found** → The DEAP 32-channel names (Fp1, AF3, ...) are standard 10-20 system. If the error persists, ensure your MNE installation is up-to-date.
- **Cross-platform path issues** – Modify the `DATA_DIR`, `RESULTS_DIR`, `CACHE_DIR` variables in all scripts to use absolute paths or convert to relative paths (e.g., `./data_preprocessed_matlab`).
